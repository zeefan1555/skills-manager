## 第一章：顶层指导思想

### 1. 编码前先思考

不要想当然，不要掩盖困惑，要把权衡明确说出来。

在开始实现前：

- 明确写出你的假设；如果不确定，就先问清楚。
- 如果存在多种理解方式，要把它们列出来，不要默认挑一种直接做。
- 如果有更简单的方案，要主动指出；必要时要对过度实现提出异议。
- 如果有地方不清楚，就先停下来，说明困惑点，并发起澄清。

### 2. 简单优先

只写解决问题所需的最少代码，不做任何臆测性扩展。

- 不添加超出需求范围的功能。
- 不为一次性代码提前设计抽象。
- 不加入未被要求的“灵活性”或“可配置性”。
- 不为实际上不可能发生的场景补错误处理。
- 如果你写了 200 行，但其实 50 行就够，就应该重写并简化。

### 3. 手术式修改

只改必须改的部分，只清理你自己造成的问题。

- 不要顺手“优化”相邻代码、注释或格式。
- 不要重构那些本来没坏的部分。
- 尽量匹配现有风格。
- 如果你发现了无关的死代码，可以提出来，但不要擅自删除。

判断标准只有一个：

**每一行变更都应该能直接追溯到用户当前的需求。**

## 第二章：目标到代码闭环

## Trigger

- `goal-code-loop`
- “我给你预期输入输出，你帮我生成代码”
- “根据 IDL 和预期输入输出去写代码”
- “先写测试，再把代码做到符合预期”
- “从目标出发，用单测循环把代码做出来”

## 目标

把一份“较粗的输入 / 输出预期”转成代码实现闭环：

1. 先收敛成功标准
2. 再从 IDL、现有代码和输入输出推导实现边界
3. 先生成最小测试或最小验证手段
4. 再生成最少必要代码
5. 用测试不断循环，直到输入输出符合预期

## 显式前置阶段命令

当用户明确说“现在进入某个编码阶段”时，可以先显式调用 `use-superpower`。

支持阶段：

- `write-plan`：在进入 `writing-plans` 前，先注入计划阶段注意事项
- `sub-agent`：在进入 SubAgent 开发前，先注入红绿灯测试、build 节奏和最小验证规则

`use-superpower` 只负责前置提醒与约束注入，不替代后续技能本身。

## 动态依赖的公有命令

### 默认依赖

| 当前需求 | 优先命令 | 说明 |
|---|---|---|
| 本地改码后需要验证 | `local-test` | 沿 `sh build.sh` + `./ci/run.sh` 做验证，不退化成裸 `go test` |
| 当前依赖库缺字段 / 方法 / 类型，阻塞本仓开发 | `overpass` | 先去下游仓库改动并 push，再回 `social_pet` 执行 `go get` |
| 需要改 CDS / Excel 配置 | `cds` | 配置修正 |
| 需要改 TCC / 白名单 / 开关 | `tcc` | 配置发布 |

### Overpass 触发规则

出现下面任一情况时，应主动切到 `overpass`：

1. 读代码时已经确认实现必须依赖下游新增字段、方法或接口签名
2. 编译或诊断报错显示当前依赖版本缺少目标 symbol
3. 用户明确要求“先去下游依赖库加字段，再拉回来”

不要在已经确认是下游依赖缺口时，继续在本仓硬改临时代码绕过去。

## 执行步骤

### 第 1 步：记录用户给的目标

先明确：

- 输入是什么
- 输出是什么
- 哪些行为必须满足
- 哪些行为不需要做

### 第 2 步：从 IDL 和现有代码推导实现边界

重点回答：

- 哪些类型 / 字段 / 方法签名是 IDL 已经定死的
- 哪些输出是必须稳定的
- 哪些边界条件需要测试覆盖
- 现有代码里最小改动点在哪里
- 当前依赖版本是否已经满足编码前置条件

### 第 3 步：先写测试或最小验证用例

在写实现之前，优先产出：

- 非法输入测试
- 正常输入输出测试
- 边界场景测试

如果当前仓库没有现成单测框架，就至少产出最小可运行验证脚本。

### 第 4 步：再写代码

只写让当前测试通过所需的最少代码。

不要：

- 预留没被要求的抽象
- 顺手重构相邻代码
- 一次性把未来场景都写进去

### 第 5 步：循环修正

如果测试不通过，先分类：

1. 测试写错了
2. 输入输出理解错了
3. IDL 理解错了
4. 代码逻辑错了
5. 环境 / 配置 / 依赖版本才是真问题

然后只修当前最小问题，再重新跑。

## 收口规则

只有同时满足以下条件，才算本命令完成：

- 已把高层目标改写成强成功标准
- 已有测试或最小验证手段
- 代码已经做到让当前输入输出符合预期
- 没有引入超出需求范围的额外实现
- 已确认是否需要切到 `local-test` / `overpass` / `cds` / `tcc`
