# Spec Template Reference

定义 spec 总文件夹及其内部各阶段文档的统一骨架，所有命令共享。

## 总文件夹结构

```text
{goal-slug}-spec/
├── 00-goal.md
├── 01-research.md
├── 02-options.md
└── 03-breakpoints.md
```

## `00-goal.md`

```markdown
# 目标：{目标一句话}

> 生成时间：{日期}
> 状态：进行中

## 目标

### 最终效果

### 范围
- **做**：
- **不做**：

### 约束

### 成功标准

### 用户的想法
```

## `01-research.md`

```markdown
# 阶段 1：现状调研

### 场景定义

### 当前流程

### 相关模块与代码位置

### 已知限制

### 待确认问题
```

## `02-options.md`

```markdown
# 阶段 2：方案比较与选型

### 候选方案

### 筛选过程

### 推荐方案

### 不选其他方案的理由
```

## `03-breakpoints.md`

```markdown
# 阶段 3：断点与代码改动

### 目标链路与当前链路差异

### 断点列表

### 代码改动清单

### 风险备注 / 待确认项
```

## 覆盖规则

- 同一命令重跑时，只覆盖对应阶段文件。
- 不要无故重写其他阶段文件。
- 如果前置阶段发生变化，后续阶段应提示用户是否需要重跑。
