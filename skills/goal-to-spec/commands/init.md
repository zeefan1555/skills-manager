# Init Command

> **引用规范**：`../references/spec-template.md`

## Trigger

- `init`
- `goal-to-spec`
- “帮我把目标变成 spec”
- “先初始化方案文档”
- “先把目标、范围、成功标准定一下”

## 目标

把一个模糊目标整理成可继续推进的 spec 文件夹起点，至少明确：

- 最终目标
- 范围
- 约束
- 成功标准
- 用户已有思路

## 输入

- 用户描述的目标
- 做 / 不做边界
- 技术、时间、兼容性约束
- 用户的已有方案或偏好

## 执行步骤

1. 用最少问题确认目标，不要把初始化做成长访谈。
2. 如果目标不清，优先补“最终效果”和“成功标准”。
3. 在项目根目录或用户指定位置创建一个总文件夹，例如 `{goal-slug}-spec/`。
4. 在该文件夹内先写入目标文档，例如 `00-goal.md`。
5. 不提前写阶段 1/2/3 的详细内容，只初始化总文件夹与目标文档。
6. 告诉用户下一步建议执行 `research`。

## 输出

写入总文件夹内的 `00-goal.md`，至少包含：

- `## 目标`
- `### 最终效果`
- `### 范围`
- `### 约束`
- `### 成功标准`
- `### 用户的想法`

## 返回给用户

- 已创建或初始化 spec 文件夹
- `00-goal.md` 的位置
- 已确认的目标摘要
- 还缺什么信息（如果有）
- 下一步建议：`research`

## 约束

- 不进入代码细节。
- 不提前给多个方案。
- 不直接分析断点和改动。
- 目标清楚时，最多只问 2~3 个关键问题。
