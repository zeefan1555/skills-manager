# 携程单程整页抓取 SOP

适用场景：用户要某一条单程查询结果页的**全部航班**，不是只看当前 DOM 里可见的几张卡片。

## 目标

- 打开携程单程结果页
- 如遇登录/验证码，保持同一 session 让用户处理
- 以官网结果页自身接口返回的 `flightItineraryList` 为准，整理整页全部航班
- 输出为单个 Markdown 表格

## 操作步骤

1. 先读取 `playwright-cli` skill，确认当前命令可用。
2. 用 `playwright-cli` 打开真实官网单程结果页，例如：
   - `https://flights.ctrip.com/online/list/oneway-szx-bkk0?depdate=2026-04-30&cabin=y_s&adult=2&child=0&infant=0`
3. 如果页面要求登录、短信验证、滑块或验证码：
   - 停在当前浏览器 session
   - 让用户手动完成
   - 完成后继续沿用同一个 `session/profile`
4. 在结果页加载阶段监听官网接口响应：
   - `international/search/api/search/batchSearch`
   - `international/search/api/search/pull/{searchId}`
5. 等到 `pull` 完成，或至少拿到完整 `flightItineraryList`。
6. 把页面顶部文案一起记录下来，例如：
   - `共96个航班，10个直飞`
   - `¥1629起含税价`
7. 用接口里的 `flightItineraryList` 生成整页总表，不要只靠 `.flight-item.domestic`。
8. 如用户只要整页航班表，优先整理这些字段：
   - 航空公司
   - 航班号
   - 机型
   - 出发时间 / 到达时间
   - 出发机场 / 到达机场
   - 中转数 / 经停数
   - 时长
   - 最低含税价
   - 价格选项数

## 关键判断

- **DOM 卡片数 < 页面顶部总航班数**：不要继续只滚 DOM，直接以接口列表为准。
- **顶部总数 = 接口返回数**：可以直接产出整页表。
- **顶部总数 ≠ 接口返回数**：在文档里标 `待确认`，同时写明两个数字。

## 建议输出格式

```markdown
# 携程单程机票整页采集：{出发地} → {目的地}

> 抓取时间：{时间}
> 官网：`https://flights.ctrip.com/`
> 查询条件：单程｜{出发地} → {目的地}｜{日期}｜{人数} 成人
> 页面顶部可见文案：`共XX个航班，YY个直飞｜¥ZZZ起含税价`

## 整页全部航班表

| 序号 | 航空公司 | 航班号 | 机型 | 出发时间 | 到达时间 | 出发机场 | 到达机场 | 中转 | 经停 | 时长 | 最低含税价 | 价格选项数 |
|---|---|---|---|---|---|---|---|---|---|---|---|---|
```

## 注意事项

- 数据来源必须是**用户当前打开的携程官网结果页自身响应**。
- 单程整页表适合回答“这一页所有航班都给我列出来”。
- 如果用户继续要求“每一条航班的所有套餐/舱位”，要在 itinerary 基础上继续展开 `priceList`，这是下一层原子操作。
