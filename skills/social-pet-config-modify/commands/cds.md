# CDS Command

## Trigger

- `modify`
- `cds modify`
- `config modify`
- “帮我改 cds 配置”
- “帮我去 ppe 目录改配置并提交”
- “帮我改一个 CDS 字段并给我机器人通知文案”

## 目标

在固定目录 `/Users/bytedance/bytecode/im_pet/<ppe_xxx_xxx>` 中完成一轮可追溯的 CDS 配置修改闭环：

1. 进入目标 `ppe_*` 目录
2. `svn update`
3. 搜索并定位目标配置文件
4. 只修改指定字段
5. 校验 `svn diff`
6. `svn commit`
7. 产出一条带 `@张菲帆` 的可复制提示，以及一条供用户粘贴进群的机器人通知文案

## 输入

优先收集这些输入：

- `ppe` 目录名
- 目标配置文件名、关键词或业务名
- 目标字段路径
- 新值
- commit message

如果用户只给了模糊描述，先补齐最关键缺口：

- 改哪个 `ppe`
- 改哪个配置
- 改成什么值

## 固定路径

```bash
/Users/bytedance/bytecode/im_pet/<ppe_xxx_xxx>
```

不要在其他目录里猜测或执行 `svn commit`。

## 执行步骤

### 第 1 步：进入目录并刷新 SVN

先进入目标目录，再执行：

```bash
svn update
```

约束：

- 不要在 `svn update` 之前就打开旧文件并开始修改
- 如果 `svn update` 失败，先停止并返回错误，不继续改本地脏副本

### 第 2 步：定位配置文件

优先顺序：

1. 用户直接给了文件路径：直接读该文件
2. 用户给了文件名：在目标目录内搜索同名文件
3. 用户给了关键词或业务名：在目标目录内搜索候选文件

规则：

- 如果只命中 1 个文件，继续
- 如果命中多个文件，列出候选并让用户选
- 如果 0 命中，返回“未定位到文件”，并要求用户补充信息

### 第 3 步：理解字段并做最小修改

修改前先确认：

- 字段路径是否存在
- 当前值是什么
- 新值的类型是否匹配（字符串 / 数字 / 布尔 / JSON 片段）

修改原则：

- 只改目标字段
- 不改无关格式
- 不做批量重排
- 不顺手修别的问题

如果配置是结构化文本或表格导出文件：

- 先定位具体键、行、列或片段
- 只在必要位置改值
- 修改后重新读取目标片段，确认写入成功

### 第 4 步：校验 diff

提交前必须检查：

```bash
svn diff <target-file>
svn status <target-file>
```

判定标准：

- diff 只包含目标字段的必要修改
- 没有混入无关文件
- 变更内容可直接解释

如果 diff 出现无关改动：

- 停止继续
- 明确告诉用户当前工作副本不干净，等待用户决定如何处理

### 第 5 步：提交 SVN

最小标准：

```bash
svn commit <target-file> -m "<commit-message>"
```

commit message 建议至少包含：

- 配置文件名
- 修改字段
- `ppe` 环境名

未成功拿到 `svn revision` 前，不要宣称配置已完成。

### 第 6 步：生成通知文案

提交成功后，固定生成两段文本：

1. 当前对话中返回给用户：

```text
@张菲帆 已完成 <ppe_name> 配置提交，文件: <config-file>，svn revision: <rev>
```

2. 供用户复制到 Lark 群 `oc_922afac84eeaccc771ee7a5b059e1edb` 的文案：

```text
@game_cds配置机器人 更新 <ppe_name>
```

注意：

- 这里的第二段是“可复制文案”，不是默认代替用户真正发送群消息
- 如果用户要求，也可以把两段合并成连续返回，但要保持机器人命令单独一行，便于复制

## 推荐返回结构

- `ppe` 目录
- 修改文件
- 修改字段
- 修改前值 / 修改后值
- `svn diff` 校验结果
- `svn commit` 结果
- `@张菲帆` 提示文案
- 群机器人文案

## 常见错误

- 没先 `svn update` 就开始改
- 没定位到唯一文件就直接猜
- 一个请求里改了多个不相关字段
- `svn diff` 已经混入无关改动还继续提交
- commit 成功前就通知用户“已经改好了”
- 只说“去通知机器人”，却没给出可复制文案
- 把目标环境写成了错误的 `ppe` 名称

## 收口规则

只有同时满足以下条件，才算本次命令完成：

- 已在目标 `ppe_*` 目录执行 `svn update`
- 已定位并修改正确配置文件
- `svn diff` 可解释且最小化
- `svn commit` 成功并拿到 revision
- 已返回带 `@张菲帆` 的提示文案和机器人通知文案
